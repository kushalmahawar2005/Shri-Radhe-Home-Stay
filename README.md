# Shri Radha Home Stay — Website

A fast, fully‑static marketing website for **Shri Radha Home Stay**, Nathdwara
(Rajasthan) — a homestay beside the Shrinathji Temple.

> _Stay With Peace & Devotion._

Built with **Next.js 14 (App Router) + TypeScript + Tailwind CSS + shadcn/ui +
Framer Motion**. It exports to **plain static HTML** (`output: 'export'`), so it
is truly serverless and deployable to any CDN (Vercel, Netlify, GitHub Pages,
S3/CloudFront, …). No database, no API server, no secrets required.

---

## ✨ Features

- One long landing page with smooth‑scroll section anchors
- `/rooms` and `/contact` sub‑routes (also static)
- All bookings via **WhatsApp deep links** + **tel:** links (no backend)
- Optional contact form (Formspree) that **falls back to WhatsApp** if not set
- Pichwai/heritage design system, all decorations are **inline SVG**
- SEO: metadata, Open Graph/Twitter, `sitemap.xml`, `robots.txt`, favicon,
  **JSON‑LD `LodgingBusiness`** schema with real NAP info
- Accessible (landmarks, alt text, focus states, reduced‑motion) and responsive

## 🚀 Quick start

```bash
npm install
npm run dev        # http://localhost:3000
```

### Build the static site

```bash
npm run build      # outputs to ./out  (static HTML/CSS/JS)
npm run serve      # preview the ./out folder locally
```

Deploy the `out/` folder to any static host. On **Vercel**, just import the repo
— `vercel.json` is already configured (`outputDirectory: out`).

## ✏️ Editing content (no coding needed)

**Everything** — name, phone numbers, address, links, rooms, facilities, nearby
places, testimonials — lives in **one file**:

```
lib/site-config.ts
```

Edit the values there and re‑run `npm run build`. You usually never need to
touch any component.

### A few things to set

1. **Google Maps embed** — open Google Maps → find the homestay → _Share_ →
   _Embed a map_ → copy the `src` URL → paste into
   `siteConfig.links.mapsEmbed`. Also update `mapsDirections`.
2. **Exact coordinates** — update `address.lat` / `address.lng` (used by the
   JSON‑LD schema) with the precise location.
3. **Instagram / Facebook** — already set to `@shriradhehomestay_`; add the real
   Facebook URL in `links.facebook`.

## ✉️ Optional contact form

The site works **fully without a form** — by default the contact form opens
WhatsApp with a pre‑filled message.

To collect emails instead, use **[Formspree](https://formspree.io)** (no server
needed for static export):

1. Create a form on Formspree and copy its endpoint URL.
2. Copy `.env.example` to `.env.local` and set:
   ```
   NEXT_PUBLIC_FORMSPREE_ENDPOINT=https://formspree.io/f/xxxxxxx
   ```
3. Rebuild. The form now submits to Formspree (and still offers WhatsApp).

> Prefer email via Resend? That needs a server runtime (a Route Handler), which
> means **not** using `output: 'export'`. Keep `RESEND_API_KEY` server‑side
> (never in `NEXT_PUBLIC_*`). For a purely static site, Formspree is simplest.
